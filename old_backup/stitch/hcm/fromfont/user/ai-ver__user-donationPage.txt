섹션 A. 히어로 & 기부처 안내(읽기 전용)
구성: 좌측 기부처 이미지(로고+대표 사진), 우측 텍스트(기부처 설명)
포맷:
타이틀: “기부 안내”
서브: “투명한 절차로 안전하게 전달합니다.”
본문: 기부처 미션/사용처/필요 물품 유형(요약 3~4줄)
이미지: 16:9 카드, 둥근 모서리, 얇은 보더, 라이트한 그림자
CTA(내비): 아래 폼 영역으로 스크롤 이동 버튼 “지금 기부 신청”

섹션 B. 기부 문서 양식(정보 카드 3개, 읽기 전용)
카드1: 절차(아이콘 ① 신청 → ② 검수 → ③ 포인트 지급)
카드2: 기준(수량·상태·카테고리 가능/불가 예시)
카드3: 문의(운영 시간, 이메일/카톡 채널)

섹션 C. 기부 신청 폼
폼 헤더: “기부 신청서” + 보조문구 “정확한 정보를 입력해주세요.”
필드 그룹화
기부자 정보
기부자 이름 (input, 필수)
기부 물품
카테고리 (select, 필수)
수량 (number input, 필수, 최소 1)
카테고리 별 물품 가액 (read-only, 카테고리 선택 시 자동 노출)
예상 지급 포인트 (read-only, [가액×정책] 계산값)
반송 주소(거절 발생 시)
안내 문구(읽기 전용): “검수 불가 시 반송됩니다. 반송비 규정 확인 필수.”
우편번호 자동검색 버튼 (카카오 API)
기본주소 (read-only, API 선택값)
상세주소 (input, 필수)
안내 및 약관
“안내 및 약관 보기”(텍스트 링크 → 모달 오픈, 읽기 전용)
“[ ] 약관에 동의합니다”(체크박스, 필수)
제출
“제출하기” 버튼(비활성화 기본 → 필수값 충족 & 동의 체크 시 활성화)

섹션 D. 제출 결과 상태
성공: “신청이 완료되었습니다. 검수 결과는 마이페이지에서 확인 가능”
실패: 입력/네트워크 에러 별도로 카드 안내 + 재시도/문의 버튼
2 컴포넌트·UX 디테일
입력 컴포넌트
입력 높이: 48px / 라운드 12–16px / 내부 패딩 좌우 14–16px
포커스: 메인그린 아웃라인 + 은은한 글로우
에러 상태: 보더 포인트코랄(RGB(251,110,82)), 하단 헬퍼텍스트(작은 폰트)
플레이스홀더: “예: 김지우”, “예: 소형가전”, “예: 2”
셀렉트(카테고리)
옵션 예시: 의류 / 소형가전 / 도서 / 유아용품 / 생활용품 …
선택 시 오른쪽에 [i] 카테고리 가이드 툴팁 버튼: 가능/불가 기준 요약
읽기 전용 필드(가액/예상 포인트)
스타일: 연한 배경(F2F6EB), 보더 점선 혹은 라이트 그레이, “계산됨” 라벨 배지
포맷: #,#00원 / #,#00P
주소 검색(카카오 우편번호)
버튼: “우편번호 찾기”(포인트 옐로우)
검색 완료 후 기본주소는 읽기 전용 라인에, 상세주소 별도 인풋
주소 블록 우측 상단에 “재검색” 작은 링크
체크박스 & 제출
체크박스 라벨: “(필수) 안내 및 약관에 동의합니다.”
제출 버튼: 기본 비활성(40% 불투명), 활성 시 메인그린 솔리드, 호버-딥그린
3 모달(안내 및 약관)

본문(스크롤):
수거/검수/반송 기준(불가 사유 예시 포함: 위생/파손/위조/개인정보 포함물품 등)
포인트 산정/지급 시점(검수 합격 후 영업일 X일 내 지급)
